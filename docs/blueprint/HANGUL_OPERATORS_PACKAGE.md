# 한글 연산자 패키지 (Γ) 정의 고정안

> 목표: 병렬/여백/중첩을 다루는 한글 연산자의 대수 구조와 공리를 명확히 고정해 “증명만 하면 되는” 상태로 끌어올린다.

## 1. 연산자 구성요소
- 자모 전표(초/중/종성): 사용 가능/예약 자모를 구분, 각 자모에 타입과 쓰임 범위(해석/위상/동역학/여백/논리) 지정. HS(고교)용 서브셋 표시.
- ∇_hangul (다성분 기울기): `(∂_space, ∂_time, ∂_margin, ∂_boundary, …)` 튜플. 초/중/종성 조합별 대응관계 표를 완성(예: ㅏ=공간, ㅕ=시간, ㆁ=여백 주석).
- ⊗_par (병렬): 병렬 계산 연산. 중첩(⊙), 곱(×)과의 교환/비교환 조건을 명시.

## 2. 대수 구조 (결합/교환/단위원)
- ⊗_par: 결합성, 단위원(있다면 e_par), 교환성 여부를 명시. 여백/경계 주석 충돌 시 결정 규칙(우선순위/배제/삭제) 고정.
- ⊙ (중첩), × (곱): ⊗_par와의 교환 법칙(가능/불가/조건부) 명시.
- 병렬 필터: must/optional/forbidden 주석 키를 정의, ⊗_par 후 주석을 읽어 실제 실행 체인을 결정하는 결정론적 규칙 명시.

## 3. 공리 (FOL 문장으로 고정)
- Γ-Total: 모든 well-typed 입력에 대해 정의됨.
- Γ-Type: 재작성/병렬/중첩 후 타입 보존.
- Γ-NF: 정규형 존재 및 (필요시) 유일성.
- Γ-Closed: 닫힌 연산(필수 도메인 제약 준수).
- Γ-Consv: ZFC 해석에 대해 보수성(Γ-증명 가능 ⇒ ZFC-증명 가능) 정리 목표.
- 필요한 재작성 규칙 리스트를 명시(예: 받침 이동, 병합/분해, 병렬 전개).

## 4. 종료성/합류/정규형
- 종료성(termination) 또는 표준형 환원 절차의 수렴을 위해 필요한 레마(Subject Reduction, confluence lemma 등) 목록을 고정.
- Church–Rosser(합류성) 정리 목표와 가정(critical pair 조건 등) 명시.

## 5. HS(고교) 축소판
- HS 문제에 쓸 수 있는 최소 성분/자모/규칙 서브셋을 표로 명시(예: 공간/시간/여백 3성분, 제한된 자모 집합).
- HS용 동치 정리: “HS 서브셋을 쓰는 Γ-풀이 ↔ 기존 교과서 풀이” 동치 정리의 statement(조건: 다항/유리/삼각 다항식 등).

## 6. Lean/코드 매핑
- Hangul token → Lean namespace/함수 이름 매핑표를 만든다(전체/HS 서브셋 구분).
- 기호: τ_margin, Π, K, μ 등 스펙 기호 ↔ Lean 변수/정의 이름 매핑.

## 7. 검증 체크리스트
1) 자모 전표에 미정 없음, 사용/예약 구분 완료.  
2) ∇_hangul 성분 표와 자모 대응관계 완성.  
3) ⊗_par/⊙/× 대수 구조(결합/교환/단위원/충돌 규칙) 명시.  
4) Γ-공리를 FOL 문장으로 명시, 필요한 재작성 규칙 리스트 고정.  
5) 종료성/합류/정규형 레마 목록 고정.  
6) HS 서브셋 표와 HS 동치 정리 statement 고정.  
7) Hangul→Lean 매핑표 작성.
