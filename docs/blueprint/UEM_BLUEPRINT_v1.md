# UEM Blueprint (Spec · Philosophy · Proof Roadmap)

> 목적: 흩어져 있는 스펙/철학/계획을 한 곳에 모아, Lean 포멀화의 목표와 증명 절차를 명확히 한다.  
> 상태: 정리본 v1 (구현은 최소 골격 상태, 본 청사진에 맞춰 강화 필요)  
> 범위: UEM 전체 (9차원 인식 좌표계, 여백 사영, 객체 대수, 메타 규율)

## 0. 깊이·고도화 강제 규약 (Anti-Shallow Rules)
- **최소 깊이 요건**: 모든 정의는 (i) 타입/구조, (ii) 연산/작동 규칙, (iii) 불변량/측도/손실함수, (iv) 검증 경로(정리·증명) 네 축을 명시해야 한다. 하나라도 빠지면 불수용.
- **보수성·비충돌**: 기존 수학/논리 공리 위에서 동작하도록 기저 필드/공리 집합을 명시한다. 새로운 trusted axiom 금지, 필요 시 정의나 정리로 재구성.
- **여백 보존 정량화**: 여백으로 보낸 정보는 로그/엔트로피/측도로 계량화해야 하며, `I_total = I_keep + I_margin` 형태의 보존 정리를 목표로 한다.
- **사영 왜곡 추적**: 차원 축소·사영 시 `LossMetric`을 정의하고, 증명에서 사용된 단순화/무시 항목을 기록하는 `ProjectionHistory`를 남긴다.
- **AI-평준화 금지**: 단순 유추·서술만 반복하는 출력을 거부한다. 모든 개념을 구체 타입/정리/증명 의무와 함께 제시한다.

## 1. 시스템 핵심 선언
- 총 상태 공간: `X_total = X_phys × X_rec`, 여기서 `X_rec = Π d : Dimension, Coord d` (차원별 종속 곱).
- 여백 분해: 모든 문제는 `V = V_keep ⊕ V_null` 직합으로 표현하고, 여백 사영 `Π_null : V → V_keep`은 선형/사상적 성질과 보존 채널(`margin`)을 가져야 한다.
- 여백 보존: 제거된 `V_null` 정보는 버려지지 않고 `M`에 기록/주석된다.
- 호환성: 기존 수학/논리와 보수적 호환을 유지하며, 새로운 신뢰 공리(trusted axiom)는 도입하지 않는다.

## 2. 철학 및 인식 원칙
- **한계 인식**: 필요한 최소 차원만 남기고 나머지는 여백 처리한다.
- **보수적 확장**: 기존 이론을 부정하지 않고, 필요시 메타 레이어에서 전이 규정을 명시한다.
- **시간·맥락 의존성**: `Time`과 `Margin`은 단순 `Unit`이 아닌 구조를 지닌다(순서/측도/주석).
- **비관측 대상 취급**: 직접 서술 불가한 성분은 여백에 기록하고, 연산/증명에서 명시적으로 분리한다.

## 2. 연구 프로그램 & 방향성
- **정의 심화**: 각 차원/객체에 위상·측도·모달/확률/카테고리 구조를 부여하고, 호환성/독립성 정리를 제시.
- **정량 모델**: 여백 엔트로피, 사영 손실, 인식적 연결성(양자/우주 스케일) 등을 정량화하는 함수와 불변량을 정의.
- **검증 절차**: (i) 모델 가정 → (ii) 여백화 → (iii) 핵심 차원 증명 → (iv) 손실/보존 체크 → (v) 사례 검증 순으로 표준화.
- **스케일 확장**: 힐베르트 공간(양자), 대역적 시간/위상(우주 스케일), 모달/가능세계(논리) 계층을 연결하는 사다리 정리들을 추가.
- **비관측 인식 그래프**: DAG/branch archive 모델로 병렬 사고와 반사실적 여백을 형식화, 삭제 대신 아카이브 규칙을 강제.

## 2-1. 설계 심화(증명 전 단계에서 더 깊게 고정할 항목)
- **차원/좌표 구체화**:  
  - Time: 정렬된 군(ℤ 또는 ℝ) + σ-대수/측도(국소/전역 선택).  
  - Ontic: [0,1] ⊂ ℝ의 폐구간, 순서·측도·위상 포함.  
  - Logic: 4값 격자/헤이팅 대수; 모달 연산과 결합 가능하게 설계.  
  - Modality: 가능한 세계 집합 + 접근성 관계(R)와 모달 연산(□,◇), 정규성 공리 선택 여부 명시.  
  - Agency/Value/Repr/Limit: 에이전트·역할 타입, 효용/규범 쌍, 언어/이론 표상 타입, 메타-깊이/경계 타입을 구체 타입으로 지정.  
  - `X_rec`에 대한 독립성/교환성 정리: 좌표별 업데이트/투영이 서로 간섭하지 않음을 lemma로 고정.
- **여백/손실 구조**: margin을 `(source, context, value, weight, entropy)` 로그로 모델링하고, projection loss 메트릭 `L_proj` 정의. 보존식 `I_total = I_keep + I_margin`을 목표 정리로 명시.
- **사영/직합**: `V = V_keep ⊕ V_null`, `Pi_null`을 선형사상 + margin 로그와 짝지어, ker/range/보존성을 정리로 고정.
- **병렬/반사실적 그래프**: 상태를 DAG로 표현하고, `ActiveBranch`(선형 서술)와 `Archive/Counterfactual`(비선형 병렬)을 분리. 삭제 금지, 아카이브 규칙과 사영-그래프 교환성 정리를 설계 단계에서 명시.
- **객체 대수**: Sparke/Actyon/Escalade/Secare에 시간/측도/여백/흐름 불변량을 포함시켜 정의. Sparke는 유한/측도 조건이 붙은 support, Actyon은 멀티셋과 가중 합, Escalade는 흐름 `f`와 불변량 조건, Secare는 경계/폐쇄 조건을 설계 레벨에서 서술.
- **호환성/보수성**: 새 구조가 기존 수학(모듈/측도/모달/카테고리)과 충돌하지 않도록 기저 공리(semiring/field, σ-대수, 접근성 관계의 성질 등)를 명시하고, trusted axiom을 배제.  
- **사례 준비**: 양자 얽힘(힐베르트 공간 + 모달 연결성), 모달/카테고리 사례, 뉴턴 항등식/해석학 극한 사례를 “설계 수준”에서 어떤 구조/정리를 써서 증명할지 청사진에 적시.

### 2-2. 물리/기초입자 심화 (중성자·힘 연결성 포함)
- **입자 명명/논리 오류 방지**: 이름이 주는 성질과 수학적 정의를 분리하여, 중성자(neutron)도 “전하 0, 질량 m, 스핀 1/2” 등 관측가능 좌표로만 정의하고, 이름에 따른 암묵 속성을 금지. 정의는 `(mass, charge, spin, isospin, context)` 튜플로 표준화.
- **힘 연결성(중력-전자기력) 탐색**:  
  - 공통 구조: 게이지/대칭/보존법칙을 모듈/섬유다발/연결(접속)으로 표현.  
  - 중력: 리만 다양체 + Levi-Civita connection + curvature; 에너지-모멘텀 텐서를 source로.  
  - 전자기: U(1) 연결/곡률로 표현; 맥스웰 방정식.  
  - 연결성 가설: “정보/여백 보존” 관점에서 두 장의 잠재적 공통 불변량(엔트로피 흐름, 게이지-위상 불변량)을 정의하고, 사영/여백 로그로 비교하는 정리 목표를 문서화. (예: 커브처 스칼라/게이지 곡률의 통합 가능성 여부를 여백에 기록; 무모순 조건을 정리로 시도)
- **양자 터널링/스펙트럼**: Hilbert 공간 H=L²(ℝ), Hamiltonian, 장벽 퍼텐셜, 전이/반사 계수를 `V_keep`(관측)과 `V_null`(버려지는 위상/경로)로 분해하여 사영 보존/손실을 정리 목표로 명시.
- **혼돈/난류/비선형**: 삼체/난류를 위해 초기조건 감도(Lyapunov), Reynolds 분해 등을 사영과 여백 로그로 모델링; 불변량(에너지/각운동량/평균 방정식) 보존을 정리 목표로 명시.

### 2-3. 고차 논리/객체/연산 심화 (빈틈 채우기)
- **기초 논리/형식**: 모달/다치/고차 논리 기반, dependent type/HoTT 친화적 코어를 전제로 하며, trusted axiom 금지. 기저 공리(semiring/field, σ-대수, 위상/측도, 모달 접근성 관계 등)를 명문화하고 모든 확장은 이에 보수적이어야 함을 선언.
- **고차 객체**: 스칼라/벡터/텐서를 넘어 섬유다발/연결/곡률, n-카테고리/호모토피/토포스적 경로/동등성 개념을 설계 레벨에 포함. 여백 로그/손실 메트릭을 고차 객체에도 적용 가능하도록 요구.
- **한글 연산자/기호 지침**: 한국어 서술과 수학 기호가 혼재할 때, 모든 연산자는 수학 기호를 우선 표기하고, 한글은 주석/설명에만 사용. 여백/중첩/압축동역학/차원축소 등의 용어는 대응하는 수학 연산(사영, 직합, 커널/상, 로그 집계)과 함께 정의문을 요구.
- **여백 엔트로피/손실 메트릭**: margin을 `(source, context, value, weight, entropy)`로 모델링하고, 사영 손실 `L_proj`를 정의하여 `I_total = I_keep + I_margin`을 설계 목표로 고정. 모든 사영/압축/차원축소 연산은 이 로그를 갱신해야 함을 규약화.
- **병렬/반사실적 그래프**: 상태를 DAG/Archive로 정의, 삭제 금지, 반사실적은 Archive 파티션으로 이동. 사영-그래프 교환성, 병합/분기 일관성을 정리 목표로 명시.
- **객체 대수(고정 불변량)**: Sparke/Actyon/Escalade/Secare는 시간/측도/여백/흐름 불변량을 필수 필드로 포함. AddCommMonoid/SMul, 흐름-사영 교환성, 평균/플럭추에이션 분해(Reynolds) 시 불변량 보존을 정리 목표로 명시.
- **보수성/무충돌 정리 목표**: 모든 새 구조가 기존 수학/논리와 모순되지 않음을 lemma/정리로 문서화(기저 공리 집합 위에서). trusted axiom 추가 금지 재확인.
- **사례 최소 증거 세트**: 삼체(에너지/각운동량 보존 + 여백 로그), 양자 터널링(전이/반사 계수 + 여백), 난류(Reynolds 분해 사영 일관성), 모달/카테고리 사례(자연 변환/정보 보존), 뉴턴 항등식/해석학 극한 사례를 “어떤 구조/정리로 증명할지” 설계 단계에서 명시.

### 2-4. 실행 우선순위(증명 전 준비를 더 단단히)
1) X_rec 보강: 모달/측도/위상/에이전시 규칙과 투영/업데이트/독립성 lemma.  
2) Margin/L_proj: 로그 갱신 규약 + 보존식(`I_total`) 목표 정리.  
3) Graph↔Projection: 분기/병합/Archive 규칙, 사영-그래프 교환성 정리.  
4) Ext 객체 불변량: AddCommMonoid/SMul, 흐름-사영 교환성, Reynolds 보존.  
5) 사례 스켈레톤: 삼체, 터널링, 난류, 모달/카테고리, 뉴턴/해석학 사례용 정리 초안.  
6) 보수성 정리: 기저 공리 위 무모순·보수성 lemma.

## 3. 형식 스펙 (Lean 타깃 구조)
### 3.1 차원/좌표
- `inductive Dimension` (d1–d9) + `Coord : Dimension → Type`에 각 구조 부여:
  - 예: `Ontic : {x : ℝ // 0 ≤ x ∧ x ≤ 1}`, `Logic : Fin 4` 또는 사용자 정의 격자, `Modality`는 모달 연산을 갖는 자료형 등.
- `X_rec := (d : Dimension) → Coord d`; 투영/삽입, 기본 정리(점별 동일성, 업데이트 렘마) 제공.

### 3.2 여백 사영 (직합)
- `V := V_keep ⊕ V_null` (모듈/선형공간 가정).
- `Π_null : V →ₗ[R] V_keep`, `inc_keep : V_keep →ₗ[R] V`, `inc_null : V_null →ₗ[R] V`.
- 증명 목표: `ker Π_null = {0} ⊕ V_null`, `range Π_null = V_keep`, `Π_null ∘ inc_keep = id`, `Π_null ∘ inc_null = const 0`, idempotence, 보충성.
- `margin` 채널: `(v_keep, v_null) ↦ (v_keep, log v_null)` 식으로 null 성분을 주석화하는 구조체 정의.

### 3.3 객체 대수
- `Sparke`: `{X : Val, support : Set Time, margin : Margin}`; `support`는 유한/측도 가정. 덧셈은 `X` 가산 + `support` 합집합 + `margin` 병합 규칙.
- `Actyon`: 유한 멀티셋의 `Sparke` + `agent/role` 타입 + 가중치/누적 규칙.
- `Escalade`: 흐름 `f : S → S`, 시간 영역 `Set Time`, 불변량/단조성 조건.
- `Secare`: 부분공간 `S`와 경계 `boundary`가 흐름에 대해 안정적임을 요구.

### 3.4 상태
- `State R V_phys := { phys : V_phys, epistemic : X_rec, margin : Margin }`.
- 차원별 렌즈, 진화 연산이 `Π_null` 및 보존 규칙과 호환되도록 정의.

### 3.5 메타 규율 (비-axiom 지향)
- `exclusion_first`: 적합도 함수와 선택 규칙을 **정의**로 제공하고, 성질을 정리로 증명.
- `meta_bound_six`: WellFounded 재귀/높이 제한으로 구현, 신뢰 공리 사용 금지.

## 4. 증명 로드맵 (우선 P1–P4)
- **P1 (Null Projection)**: 선형성, ker/range, 직합 유일 분해, idempotence, margin 보존 정리.
- **P2 (Sparke Monoid/Module)**: 유한/측도 가정 하에서 AddCommMonoid, `nsmul`/`smul`, support 합집합의 결합/교환/보존 성질.
- **P3 (Actyon/Escalade 안정성)**: 흐름이 여백 사영과 교환(`Π_null ∘ f = f_keep ∘ Π_null`), 경계 보존.
- **P4 (Dimension 정합성)**: `X_rec` 업데이트/투영 렘마, 차원별 구획의 독립성/교환성.
- 사례 증명: 스펙의 뉴턴 항등식 예시를 Lean에서 여백화 전략으로 재구성, 최소 증거 세트 확보.
